# 미해결 과제(Open Loop) 템플릿

새 미해결 과제를 만들 때 이 파일을 복사하여 `영문-슬러그.md` 로 저장하세요.

- 파일명: `영문-슬러그.md` (예: `pay-the-debt.md`)
- 표시명은 아래 테이블에 한글로 작성

---

| 항목 | 내용 |
|------|------|
| **표시명** | (한글 이름, 예: 남겨진 빚의 정산) |
| **유형** | (목표/갈등/부채/약속/미스터리/위험요소 등) |
| **시작 화** | (예: 4화) |
| **해결 예정** | (예: 10~12화, 미정 가능) |
| **상태** | (열림/진행/해결/보류/폐기) |
| **연관 요소** | (예: `characters/han-seo-jun.md`, `hooks/mysterious-scar.md`) |
| **태그** | (예: #부채 #추적 #회수필수) |

## 한 줄 요약

- 독자가 “아직 안 끝났네” 하고 기억해야 하는 한 문장

## 현재 상태(지금 왜 열려 있나)

- 무엇이 해결되지 않았는가
- 해결을 막는 장애물/조건

## 진행(갱신 기록)

- 언제(몇 화) 어떤 변화가 있었나
- 정보가 늘었나, 악화됐나, 해결에 가까워졌나

## 해결 조건(클로저 정의)

- 어떤 장면/결정/사건이 일어나면 “해결”로 판단할지
- 부분 해결/완전 해결 구분이 필요하면 명시

## 노트

- 일관성 체크 포인트(모순 방지)
- 폐기 시 처리(독자 납득 장치)
