# 코드 개선 및 리팩토링 제안 프롬프트

## 대상 정보 (Target Information)

> [!TIP]
> 이용자가 아래 내용을 직접 수정하여 작업 대상을 지정할 수 있습니다.

- **리팩토링 참조 파일/폴더**: `src\lib\components\docs`
- **적용할 페르소나**: @[.vibe-coding/PERSONA/A11Y_UX_COPILOT.md]

---

지정된 파일에 대해 특정 페르소나 관점에서 코드 품질을 검토하고 개선 의견을 작성하기 위한 표준 프롬프트입니다.

## 지침 (Core Instructions)

1. **관점**: 지정된 **페르소나**의 전문 지식과 가이드라인을 바탕으로 코드를 분석하십시오.
2. **검토 대상**: 요청에 명시된 파일 경로의 코드를 정독하십시오.
3. **결과 기록**: @[.vibe-coding/REQUESTS.md] 문서 하단의 **📋 작업 상세 내용** 섹션에 해당 항목을 찾아 상세 개선안을 작성하십시오.
    - **세부 작업 분할**: 리팩토링할 내용이 많거나 복잡할 경우, 한 파일에 대해서도 여러 개의 순하위 태스크로 나누어 `REQUESTS.md`에 개별 등록할 수 있습니다.
4. **출력 형식 및 상태 관리**:
    - **상태 업데이트**:
        - 분석 및 제안만 완료된 경우: `🏗️ 진행 중`으로 유지하고 비고란에 링크를 추가하십시오.
        - 실제 코드 수정(구현)까지 모두 완료된 경우에만: `✅ 완료`로 변경하십시오.
    - **상단 테이블**: '비고'란에 간결한 헤더 링크(`[상세 보기](#순번-파일명-리팩토링)`)를 추가하거나 업데이트하십시오.
    - **상세 섹션**: 헤더를 `### [순번]. [파일명] 리팩토링` 형식으로 작성하고, 내부에 대상 파일의 전체 경로와 상세 개선안을 기록하십시오.

## 작업 프로세스

- **상황 파악**: 대상 파일 목록 확인
- **코드 분석**: 페르소나 원칙에 따른 취약점/개선점 도출
- **제안서 작성**: `REQUESTS.md`에 새 태스크로 등록하고 상세 내용을 기록
- **완료 보고**: `REQUESTS.md` 등에 작업 상태 업데이트
