# typesafe-i18n: 2024-11-13 ~ 2025-11-13 업데이트 현황 및 실무 가이드

**`typesafe-i18n`** 패키지는 요청 기간(2024-11-13 ~ 2025-11-13) 동안 **새로운 버전이 단 하나도 릴리스되지 않았습니다.** 따라서 이 기간에 해당하는 **API `deprecated`, 신규 기능, 또는 사용법 변경은 존재하지 않습니다.**

이 라이브러리는 2023년 8월 릴리스 이후 사실상 **안정(Stable) 모드**에 진입했습니다.

---

## 1. 요청 기간 내 업데이트 부재 현황

| 기간 | 패키지 버전 변화 | 명시적 API Deprecation |
| :--- | :--- | :--- |
| 2024-11-13 ~ 2025-11-13 | 0 (변화 없음) | 없음 (패키지 동결 상태) |

---

## 2. 참고용: v5 메이저 버전의 핵심 구조적 변화 (2023년 이전)

비록 요청 기간 밖이지만, 현재 v5를 사용하는 개발자가 알아야 할 **설계 원칙**과 **주요 변경점**은 다음과 같습니다. 이 구조는 현재까지 유지되고 있습니다.

| 영역 | 과거 핵심 변경 (v5) | 현재 코드 설계 시 유의점 |
| :--- | :--- | :--- |
| **Generator** | **Rollup/Webpack 플러그인 기반 Generator 제거 (5.0.0)** | **CLI 기반(`npx typesafe-i18n`)**만 공식 지원합니다. CLI 마이그레이션이 필수입니다. |
| **Core API** | `typesafeI18nString/Object` 저수준 API 공개 (5.0.0) | `i18n()` 함수 래핑 외, 저수준 API를 활용하여 커스텀 툴링을 구축할 수 있습니다. |
| **유틸리티** | `isLocale`, `isNamespace` 유틸리티 도입 (5.13.0) | `string` 검증 시 배열 `includes` 대신 **표준 유틸 함수**를 사용해야, 코드를 명확히 하고 타입 가드(Type Guard) 기능을 활용할 수 있습니다. |
| **딕셔너리 확장** | `extendDictionary` 함수 추가 (5.23.0) | 지역 기반 변형(en-US)을 구현할 때, **`extendDictionary(base, overrides)`** 패턴을 사용하여 중첩 딕셔너리를 안전하게 병합하는 것이 권장됩니다. |
| **모듈 시스템** | **다중 Adapter 지원** (5.7.0), **ESM 지원 개선** (5.17.0) | React, Svelte, Vue, Solid, Deno 등 여러 런타임을 포함하는 프로젝트는 `adapters` 배열 설정으로 단일 설정 파일을 공유할 수 있습니다. |

---

## 3. 실무적 결론 및 전략

**`typesafe-i18n`**은 현재 API가 매우 안정적이며, 새로운 기능을 적극적으로 추가하지 않는 대신 기존 기능의 타입 안전성과 호환성을 유지하는 **보수적인 전략**을 취하고 있습니다.

1. **리스크:** 패키지 동결 상태이므로, Svelte 5나 최신 TS/Bundler 환경에서 예기치 않은 타입 호환성 문제가 발생할 경우 **자체적으로 해결**해야 할 수 있습니다.
2. **전략:** API가 안정적이라는 장점을 활용하여, **`isLocale` 유틸리티** 및 **`extendDictionary`** 같은 고급 유틸 함수를 적극적으로 도입해 코드의 품질과 유지보수성을 높이는 방향으로 리팩토링하십시오.
