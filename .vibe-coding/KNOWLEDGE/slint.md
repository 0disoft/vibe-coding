# Slint 아키텍처 리포트: v1.9 - v1.14 핵심 변경 사항 분석

2024년 말부터 2025년 10월까지 Slint의 발전은 **'데스크톱 UX 완성도 향상'**과 **'DSL 표현력 강화'**에 집중되었습니다. 특히 메뉴 시스템의 네이티브화, WGPU 통합, 그리고 새로운 그래픽 변환 체계 도입은 단순한 기능 추가를 넘어 아키텍처 수준의 변화를 요구합니다.

본 문서는 GitHub 릴리즈 노트와 공식 기술 블로그를 기반으로, 실제 코드베이스에 영향을 미치는 중요 변경 사항만을 선별하여 분석합니다.

---

| **v1.10** | '25.02      | **Desktop UX:** 네이티브 메뉴 시스템(`MenuBar`) 도입. 임베디드용 SDF 폰트 최적화.                             |

## 2. Critical Changes: 코드베이스 영향도 분석

### 2-1. 배포 및 윈도우 제어 (v1.9)

- **번역 번들링 (Translation Bundling):** `.mo` 파일을 별도로 배포하던 방식에서 실행 파일 내장(Bundle) 방식으로 전환하십시오. `SLINT_BUNDLE_TRANSLATIONS` 옵션을 통해 단일 파일 배포의 이점을 취할 수 있습니다.
- **팝업 정책 (Popup Policy):** `click-on-close` 속성은 레거시가 될 가능성이 높습니다. `close-policy` 속성을 사용하여 '외부 클릭 시 닫기' 등 명확한 닫기 전략을 수립하십시오.
- **백엔드 강제 (Backend Selector):** 소프트웨어 렌더러 강제 등 특정 렌더링 백엔드가 필요한 경우, `BackendSelector` API를 통해 초기화 단계에서 명시적으로 제어할 수 있습니다.

### 2-2. 데스크톱 네이티브 경험 (v1.10 - v1.11)

- **메뉴 시스템 (Menu System):** 기존에 위젯으로 구현했던 커스텀 메뉴를 `MenuBar`, `Menu`, `MenuItem` DSL 요소로 전면 교체하십시오. 이는 OS(Windows/macOS) 네이티브 메뉴와 통합되어 훨씬 자연스러운 UX를 제공합니다.
- **로직의 DSL 이동:** `to-fixed()`, `to-uppercase()` 등의 문자열 함수가 DSL에 추가되었습니다. 단순 포맷팅을 위해 Rust/C++에서 가공된 문자열을 넘겨주던 코드를 제거하고, 뷰(View) 레벨에서 처리하도록 리팩토링하십시오.

### 2-3. 그래픽스 및 통합 (v1.12 - v1.14)

- **WGPU/Bevy 통합:** Rust 환경에서 3D 렌더링이 필요한 경우, 별도 창을 띄우는 대신 Slint 내부에 WGPU 컨텍스트를 삽입하는 아키텍처를 채택하십시오. UI와 3D 콘텐츠의 매끄러운 통합이 가능해집니다.
- **Universal Transforms (v1.14):** 개별적인 `rotation`, `scale` 속성 사용을 지양하고, 통합된 변환 시스템(Transform System)을 사용하도록 코드를 점검해야 합니다. 특히 복잡한 애니메이션 구현 시 변경된 좌표계 동작을 확인하십시오.
- **텍스트 엔진 교체:** Fontique/Parley 기반의 통합 텍스트 엔진이 도입되었습니다. 커스텀 폰트나 다국어(CJK, RTL) 레이아웃이 포함된 화면은 반드시 시각적 회귀 테스트(Visual Regression Test)를 수행하여 레이아웃 깨짐 현상을 방지하십시오.

### 2-4. 개발 생산성 및 언어 기능 (v1.13)

- **라이브 프리뷰 (Live Preview):** `SLINT_LIVE_PREVIEW=1` 환경 변수를 CI/CD 파이프라인이나 로컬 개발 스크립트에 통합하십시오. 앱 재컴파일 없이 UI 수정 사항을 즉시 확인할 수 있어 개발 속도가 비약적으로 향상됩니다.
- **로컬 변수 (`let`):** 임시 계산값을 전역 프로퍼티(Property)로 선언하던 관행을 버리고, `let` 키워드를 사용하여 변수의 스코프를 제한하십시오. 이는 상태 관리의 복잡도를 낮추고 사이드 이펙트를 줄이는 데 기여합니다.
- **키 이벤트 캡처 (Key Capture):** `FocusScope`의 캡처 단계를 활용하여 글로벌 단축키나 입력 가로채기 로직을 구현하십시오. 기존의 복잡한 이벤트 버블링 제어 코드를 단순화할 수 있습니다.

## 3. Action Plan: 마이그레이션 전략

1. **v1.10 타겟 리팩토링:** 데스크톱 앱이라면 최우선적으로 메뉴 시스템을 `MenuBar` DSL로 교체하여 네이티브 룩앤필을 확보하십시오.
2. **DSL 최적화:** v1.11 및 v1.13의 기능을 활용하여, 불필요한 백엔드 로직(문자열 포맷팅 등)을 제거하고 DSL 내부에서 처리하도록 코드를 경량화하십시오. `let` 사용을 표준화하여 가독성을 높이십시오.
3. **렌더링 검증:** v1.14로 업그레이드 시, 텍스트 렌더링 및 그래픽 변환 로직에 대한 전수 테스트를 수행하십시오. 특히 다국어 지원이 중요한 경우 필수 절차입니다.
4. **개발 환경 고도화:** 라이브 프리뷰 기능을 팀 내 표준 개발 워크플로우로 정착시켜 UI 개발 생산성을 극대화하십시오.

Slint는 이제 단순한 임베디드 UI 툴킷을 넘어, 데스크톱과 모바일을 아우르는 강력한 크로스 플랫폼 솔루션으로 진화하고 있습니다. 이번 업데이트들은 프레임워크의 성숙도를 보여주는 지표이며, 이를 적극적으로 도입함으로써 애플리케이션의 품질과 유지보수성을 크게 향상시킬 수 있습니다.
