# giscus: 2024년 11월 ~ 2025년 12월 주요 아키텍처 변화 및 실무 가이드

giscus는 2024년 11월 13일부터 2025년 12월 3일 사이에 **API 계약의 파괴적인 변경이나 `deprecated`된 기능을 공개적으로 릴리스하지 않았습니다.** 이 기간의 유일한 변경은 **v1.6.0**에서 진행된 **타입 정의 업데이트**가 전부입니다.

---

## 1. 요청 기간 내 업데이트 현황 및 결론

| 버전 / 패키지           | 릴리즈 시기 | 핵심 변경 사항                                                | 명시적 Deprecation |
| :---------------------- | :---------- | :------------------------------------------------------------ | :----------------- |
| **giscus (1.6.0)**      | 2024-12-31  | **타입 정의 동기화:** 새롭게 지원되는 언어 및 테마 목록 반영. | 없음               |
| **@giscus/vue (3.1.1)** | 2024-12-31  | 3.1.0의 빌드 문제를 수정한 안정화 패치.                       | 없음               |

**결론:** 이 기간 동안 giscus는 API가 동결된 상태입니다. 모든 변화는 **TypeScript 타입 지원 범위 확장**에 국한됩니다.

---

## 2. “실제 코딩”에 미치는 영향

### 2-1. API 및 사용법 안정성

- **호환성 유지:** 기존에 사용하던 모든 **필수 Props**(`repo`, `repoId`, `category`, `mapping` 등)의 이름, 시그니처, 런타임 동작은 **v1.6.0**에서도 그대로 유지됩니다.
- **타입 안전성:** TypeScript 환경에서는 **v1.6.0 / v3.1.x**로 업데이트 시, 새로 추가된 언어 코드나 테마 이름을 입력해도 타입 검사를 통과하게 되어 개발 경험이 개선됩니다.
  - **Action:** 기존 코드를 수정할 필요는 없으며, 새로운 테마(예: `transparent_dark`, `preferred_color_scheme` 등)를 사용할 때 TS의 자동 완성 지원이 향상됩니다.

### 2-2. 마이그레이션 주의사항 (v3.1.1 선택)

- **Vue 래퍼:** `@giscus/vue`를 사용하는 프로젝트는 **v3.1.1** 버전을 사용하십시오. v3.1.0은 빌드 문제가 있어 사용이 권장되지 않습니다.

---

## 3. 참고: 기간 직전의 중요 브레이킹 변경

요청 기간 바로 이전에 (2024년 초, v3.0.0) 발생한 **ESM-only 전환**이 현재에도 가장 큰 구조적 제약으로 작용합니다.

- **ESM-Only:** `giscus` 및 모든 프레임워크 래퍼 패키지(`@giscus/react`, `svelte` 등)는 **CommonJS 지원을 중단**하고 ESM 전용으로 전환되었습니다.
- **Action:** 현재도 `require('@giscus/react')`와 같은 CommonJS 구문을 사용하는 프로젝트는 **3.1.x 버전으로 업그레이드하기 전에 반드시 `import` 구문 기반의 ESM 빌드 체계로 전환**해야 합니다.

---

## 4. 최종 결론

giscus를 사용하거나 새로 도입할 때, **v3.0.0의 ESM-only 전환**만 성공적으로 완료했다면, **v1.6.0 / v3.1.x**로의 업데이트는 코드베이스에 위험을 주지 않는 안전한 품질 개선입니다.
