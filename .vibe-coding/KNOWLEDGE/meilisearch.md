# Meilisearch v1.12.0 - v1.28.0 아키텍처 리포트: AI 검색의 프로덕션 레디

Meilisearch는 2024년 말부터 2025년 말까지 **'AI 및 벡터 검색 기능의 안정화'**와 **'대용량 작업 처리 자동화'**에 집중하며 검색 엔진의 경계를 확장했습니다. 특히 v1.17.0의 Chat 설정 메서드 변경과 v1.12.0의 Batches API 도입은 클라이언트/백엔드 로직의 변화를 강제하는 주요 분기점입니다.

본 리포트는 해당 기간의 변경 사항을 분석하여, 실무에서 즉시 적용해야 할 아키텍처 전략과 마이그레이션 포인트를 제시합니다.

## 2. Critical Action Items: 코드 및 운영 정책 점검

### 2-1. API 메서드 및 호출 변경 (v1.17.0)

Chat Completions 관련 설정 엔드포인트의 HTTP 메서드가 변경되었습니다.

* **Deprecated:** `PUT /chat/settings`
* **Replacement:** **`PATCH /chat/settings`**
* **Action:** Chat 관련 설정 API를 호출하는 모든 클라이언트/백엔드 코드를 `PATCH` 메서드로 수정하십시오. `PUT`을 유지할 경우 4xx 오류가 발생합니다.

### 2-2. 대용량 작업 관리 (v1.12.0, v1.17.0)

* **Batch Processing (v1.12.0):** 수백, 수천 개의 문서를 개별적으로 추가/업데이트/삭제하는 코드를 Batches API로 전환하십시오. 네트워크 오버헤드를 줄이고, `progressTrace` (v1.20.0)를 통해 진행 상황을 세밀하게 모니터링할 수 있습니다.
* **Webhook 자동화 (v1.17.0):** 인덱스 생성 완료, 태스크 실패 등 중요 이벤트에 대해 폴링(Polling) 대신 Webhook API를 활용하십시오. 외부 URL에 이벤트를 전송하여 CI/CD나 알림 시스템을 비동기적으로 트리거할 수 있습니다.

### 2-3. 검색 결과 정합성 확보 (v1.14.0, v1.21.0)

* **Typo Tolerance 검증:** `typoTolerance: false`가 설정된 속성에서 Prefix Search의 동작이 v1.14.0에서 수정되었습니다. 해당 조합을 사용하는 검색 쿼리에 대해 **회귀 테스트**를 수행하여 결과가 의도와 일치하는지 확인하십시오.
* **벡터 스토어 실험:** v1.21.0에서 새로운 벡터 스토어 백엔드가 도입되었습니다. 벡터 검색 성능이 중요한 컬렉션이라면 `vectorStoreSetting` 실험 기능을 활성화하여 성능을 벤치마킹하고, A/B 테스트를 통해 도입 여부를 결정하십시오.

## 3. Strategic Adoption: 도입해야 할 신규 패턴

### 3-1. AI 검색 기능 확장

* **멀티모달 검색 (v1.16.0):** 임베더 설정에 이미지 모델을 추가하여 텍스트-이미지 통합 검색 기능을 구축하십시오.
* **GeoJSON 필터 (v1.22.0):** 복잡한 지리적 영역(다각형) 검색이 필요하다면, 문서에 GeoJSON 필드를 추가하고 `_geoPolygon` 필터를 활용하여 기존의 `_geoBoundingBox`보다 정교한 지역 기반 UX를 제공하십시오.

### 3-2. 운영 및 인프라 효율화

* **인덱스 관리 단순화 (v1.18.0):** 인덱스 이름을 변경할 때 더 이상 재색인할 필요 없이 인덱스 rename API를 사용하십시오. 마이그레이션 및 테스트 환경의 관리 코드를 단순화할 수 있습니다.
* **디버깅 지원:** 검색 결과에 임베딩 벡터와 쿼리 벡터를 포함하는 옵션(`retrieveVectors`, `queryVector`)을 활성화하여 벡터 기반 재랭킹 로직이나 디버깅 파이프라인을 구축하십시오.

## 4. Conclusion

Meilisearch v1.12~1.28 구간은 **"AI 검색의 프로덕션 레디"**를 목표로 하였습니다. 이번 마이그레이션의 핵심은 **v1.17.0의 메서드 변경 반영**과 **Batches API를 통한 인덱싱 효율화**입니다. 신규 GeoJSON 필터링 및 Webhook 자동화를 통해 사용자 경험과 백엔드 운영 효율성을 동시에 개선하십시오.
